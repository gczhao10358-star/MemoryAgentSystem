# 智忆助理 - 飞书接入指南

## 概述

本文档指导你如何在智忆助理中配置飞书机器人，实现通过飞书接收提醒消息。

```
智忆助理设置提醒 → 时间到达 → 飞书推送消息
```

## 一、创建飞书应用

### 1. 访问飞书开放平台

打开 [https://open.feishu.cn/app](https://open.feishu.cn/app) 并登录

### 2. 创建企业自建应用

1. 点击「创建企业自建应用」
2. 填写应用名称：如「智忆助理」
3. 填写应用描述：个人提醒助手
4. 点击「确定创建」

### 3. 获取应用凭证

进入应用详情页 → 「凭证与基础信息」：

```
App ID: cli_xxxxx (复制)
App Secret: 点击显示并复制
```

### 4. 添加权限

进入「权限管理」，添加以下权限：

- `im:message:send` - 发送消息
- `im:message:group_msg` - 发送群消息
- `contact:user.base:readonly` - 读取用户基本信息

点击「批量开通」。

### 5. 发布应用

进入「版本管理与发布」：

1. 点击「创建版本」
2. 填写版本号：1.0.0
3. 填写更新说明：初始版本
4. 点击「保存」
5. 点击「申请发布」
6. 联系企业管理员审核通过

### 6. 设置可见范围

进入「应用功能」→ 「基础信息」：

- 设置「可见范围」为「全部员工」或指定成员
- 确保你能看到该应用

## 二、获取接收者 ID

### 方法1：通过飞书开放平台调试工具

1. 访问 [飞书开放平台的用户身份](https://open.feishu.cn/api-explorer?api=user)
2. 选择「获取用户信息」接口
3. 点击「获取 tenant_access_token」（会自动使用你的应用）
4. 填写参数获取用户信息，找到 `open_id`

### 方法2：通过飞书机器人Webhook（推荐）

如果你只需要给自己发消息，更简单的方法是：

1. 在飞书群中创建一个机器人
2. 获取 Webhook 地址
3. 使用 Webhook 方式发送（无需App ID/Secret）

**注意：** 当前实现使用 API 方式，需要获取 open_id。

## 三、在智忆助理中配置

### 1. 启动智忆助理

```bash
cd C:\Users\78675\Desktop\记忆助理智能体

```

访问 `http://localhost:5173` 或 `http://localhost:8000/app`

### 3. 进入设置页面

点击左侧菜单「设置」

### 4. 填写飞书配置

| 字段 | 值 | 说明 |
|------|-----|------|
| App ID | cli_xxxxx | 从飞书应用凭证获取 |
| App Secret | xxxxx | 从飞书应用凭证获取 |
| 接收者 ID | ou_xxxxx | 你的飞书 open_id |
| ID 类型 | open_id | 保持默认 |

### 5. 保存并测试

1. 点击「保存配置」
2. 点击「发送测试消息」
3. 检查飞书是否收到消息

## 四、使用飞书接收提醒

### 设置提醒

和之前一样发送消息：

```bash
curl -X POST "http://localhost:8000/api/chat" \
  -H "Content-Type: application/json" \
  -d '{"user_id":"test","message":"1分钟后提醒我该喝水了"}'
```

### 接收提醒

时间到达后，你会在飞书收到消息：

```
⏰ 提醒

该喝水了
```

## 五、API 接口

### 保存配置

```http
POST /api/platform/lark/config
Content-Type: application/json

{
  "user_id": "test",
  "app_id": "cli_xxxxx",
  "app_secret": "xxxxx",
  "receive_id": "ou_xxxxx",
  "receive_id_type": "open_id"
}
```

### 测试连接

```http
POST /api/platform/lark/test
Content-Type: application/json

{
  "user_id": "test",
  "content": "测试消息"
}
```

### 获取配置状态

```http
GET /api/platform/status?user_id=test
```

### 删除配置

```http
DELETE /api/platform/lark/config?user_id=test
```

## 六、前端集成示例

```vue
<template>
  <div class="lark-config">
    <h2>飞书配置</h2>

    <input v-model="config.app_id" placeholder="App ID" />
    <input v-model="config.app_secret" type="password" placeholder="App Secret" />
    <input v-model="config.receive_id" placeholder="接收者 ID" />

    <button @click="saveConfig">保存配置</button>
    <button @click="testConnection">测试连接</button>
  </div>
</template>

<script setup>
import { ref } from 'vue'

const config = ref({
  app_id: '',
  app_secret: '',
  receive_id: '',
  receive_id_type: 'open_id'
})

async function saveConfig() {
  await fetch('/api/platform/lark/config', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      user_id: 'test',
      ...config.value
    })
  })
}

async function testConnection() {
  await fetch('/api/platform/lark/test', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      user_id: 'test',
      content: '测试消息'
    })
  })
}
</script>
```

## 七、常见问题

### Q1: 提示 "未找到飞书配置"

确保已调用 `/api/platform/lark/config` 保存配置，且 `enabled` 为 `true`。

### Q2: 提示 "发送失败"

检查：
1. App ID 和 App Secret 是否正确
2. 应用是否已发布并通过审核
3. 应用是否有发送消息权限
4. 接收者 ID 是否正确

### Q3: 如何获取自己的 open_id？

方法1：使用飞书开放平台调试工具
方法2：在应用中添加事件订阅，用户授权后获取

### Q4: 可以给群聊发消息吗？

可以！将 `receive_id` 改为群的 chat_id，`receive_id_type` 改为 `chat_id`。

### Q5: 支持其他平台吗？

目前实现了飞书，可以类似方式扩展钉钉、企业微信等平台。

## 九、安全建议

1. **App Secret 加密存储** - 生产环境应加密
2. **Token 缓存** - 已自动实现，避免频繁请求
3. **权限控制** - 只开通必要的权限
4. **HTTPS 传输** - 生产环境必须使用 HTTPS

## 1. 项目概述

### 1.1 项目简介

**智忆助理 (MemoryMate)** 是一个具备长期记忆能力的AI助手系统。它能够记住用户的信息、偏好、日程安排和重要事件，并在后续对话中利用这些记忆提供个性化的帮助。

### 1.2 核心特性

| 特性 | 描述 |
|------|------|
| **三层记忆系统** | 工作记忆 + 短期记忆 + 长期记忆 |
| **混合检索** | 稠密向量检索 + 稀疏文本检索 + RRF融合 |
| **用户画像** | 自动学习用户偏好、专业领域、交互风格 |
| **智能工作流** | 意图识别 → 时间解析 → 记忆存储/检索 |
| **记忆演化** | 记忆随时间衰减/强化，模拟人类遗忘机制 |
| **结构化存储** | 自动提取时间、地点、人物、事件等要素 |

### 1.3 技术栈

**后端:**
- Python 3.11+
- FastAPI (Web框架)
- FAISS (向量数据库)
- SQLite (元数据存储)
- DashScope/Qwen (LLM API)

**前端:**
- Vue 3 + Vite
- Element Plus (UI组件)
- Pinia (状态管理)
- Vue Router (路由)

#### 3.3.5 核心实现原理

精准定时调度器使用 `asyncio.wait_for()` 实现精确等待：

```python
async def _wait_and_execute(self, task: PreciseTask):
    wait_seconds = (task.scheduled_time - datetime.now()).total_seconds()

    if wait_seconds > 0:
        try:
            await asyncio.wait_for(
                task._cancel_event.wait(),
                timeout=wait_seconds
            )
            # 被取消
            return
        except asyncio.TimeoutError:
            # 时间到，继续执行
            pass

    # 执行回调
    await self._execute_callbacks(task)
```

#### 3.3.2 RRF (Reciprocal Rank Fusion) 算法

```python
def reciprocal_rank_fusion(dense_results, sparse_results, k=60):
    """
    倒数排名融合算法

    原理：将多个检索列表的排名转换为分数，然后累加
    公式：score = Σ(1 / (k + rank))

    优点：
    - 不需要训练，无监督
    - 对排名位置敏感，对分数不敏感
    - 可以融合任意数量的检索结果
    """
    scores = defaultdict(float)

    for rank, result in enumerate(dense_results):
        scores[result.id] += 1.0 / (k + rank)

    for rank, result in enumerate(sparse_results):
        scores[result.id] += 1.0 / (k + rank)

    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

### 3.5 记忆演化引擎

#### 3.5.1 权重演化公式

```python
# 时间衰减
weight *= (1 - decay_rate) ** hours_since_last_access

# 访问强化
weight = weight * (1 - reinforcement_rate) + 1.0 * reinforcement_rate

# 状态转换阈值
NEW → ACTIVE:      access_count >= 1
ACTIVE → REINFORCED: access_count >= 5
REINFORCED → CORE:   weight >= 0.8
ACTIVE → DECAYING:   weight < 0.3 and hours > 168 (7天)
DECAYING → ARCHIVED: weight < 0.15
ARCHIVED → FORGOTTEN: weight < 0.05
```

## 6. 关键技术实现

### 6.1 时间解析

```python
# 支持的表达
"明天下午3点"      → 2026-03-12 15:00
"后天"            → 2026-03-13
"下周三"          → 2026-03-18
"15号"            → 2026-03-15 (本月)
"3月15日"         → 2026-03-15
"周日晚上8点"     → 2026-03-15 20:00

# 解析流程
自然语言 → LLM提取 → 日期计算 → ISO格式
```

### 6.2 向量检索

```python
# FAISS索引配置
index = faiss.IndexFlatIP(dimension)  # 内积索引

# L2归一化 (用于余弦相似度)
faiss.normalize_L2(vectors)

# 相似度计算
# cosine_similarity = dot_product / (||a|| * ||b||)
# 归一化后 ||a|| = ||b|| = 1
# 所以 cosine_similarity = dot_product
```

### 6.3 内容过滤

# 闲聊模式过滤

## 10. 第三方平台集成

### 10.1 飞书平台 (Lark)

系统支持通过飞书平台发送提醒消息，实现多渠道通知。

#### 10.1.1 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    任务触发                              │
└─────────────┬───────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────┐  ┌──────────────────────┐
│   WebSocket 通知             │  │   飞书平台通知        │
│   (前端实时推送)              │  │   (第三方渠道)        │
└─────────────┬───────────────┘  └──────────┬───────────┘
              │                             │
              ▼                             ▼
        ┌──────────┐                 ┌──────────┐
        │ 前端界面  │                 │ 飞书应用  │
        └──────────┘                 └──────────┘
```

#### 10.1.2 配置说明

```python
@dataclass
class LarkConfig:
    app_id: str                # 飞书应用 App ID
    app_secret: str            # 飞书应用 App Secret
    receive_id: str            # 接收者ID (open_id/user_id/union_id)
    receive_id_type: str       # 接收者ID类型 (默认 open_id)
    encrypt_key: Optional[str] # 加密密钥（可选）
    verification_token: str    # 验证令牌（可选）
    enabled: bool              # 是否启用
```

#### 10.1.3 API 接口

```http
# 保存飞书配置
POST /api/platform/lark/config
{
  "user_id": "user123",
  "app_id": "cli_xxxxxx",
  "app_secret": "xxxxxxxx",
  "receive_id": "ou_xxxxxx",
  "receive_id_type": "open_id"
}

# 测试飞书连接
POST /api/platform/lark/test
{
  "user_id": "user123",
  "content": "测试消息内容",
  "title": "测试标题"
}

# 获取平台状态
GET /api/platform/status?user_id=user123

# 删除飞书配置
DELETE /api/platform/lark/config?user_id=user123
```

#### 10.1.4 使用流程

1. **创建飞书应用**：在飞书开放平台创建企业自建应用
2. **获取凭证**：获取 App ID 和 App Secret
3. **获取接收者ID**：使用飞书 API 获取用户的 open_id
4. **配置系统**：调用配置接口保存飞书配置
5. **测试连接**：发送测试消息验证配置
6. **接收提醒**：设置任务提醒，到期时同时收到 WebSocket 和飞书通知

-- 记忆表
CREATE TABLE memories (
    memory_id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    content TEXT NOT NULL,
    memory_type TEXT NOT NULL,
    state TEXT DEFAULT 'new',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    importance REAL DEFAULT 0.5,
    weight REAL DEFAULT 0.5,
    access_count INTEGER DEFAULT 0,
    metadata TEXT -- JSON格式
);

-- 精准任务表 (用于定时提醒)
CREATE TABLE precise_tasks (
    task_id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    content TEXT NOT NULL,
    title TEXT,
    scheduled_time TIMESTAMP,
    status TEXT DEFAULT 'pending',     -- pending/completed/cancelled/expired
    metadata TEXT                      -- JSON格式
);

-- 飞书配置表
CREATE TABLE platform_configs (
    user_id TEXT PRIMARY KEY,
    platform_type TEXT NOT NULL,       -- lark

    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_memories_user ON memories(user_id);
CREATE INDEX idx_memories_type ON memories(memory_type);
CREATE INDEX idx_memories_state ON memories(state);
CREATE INDEX idx_tasks_user_id ON precise_tasks(user_id);
CREATE INDEX idx_tasks_status ON precise_tasks(status);
CREATE INDEX idx_platform_configs_user ON platform_configs(user_id);

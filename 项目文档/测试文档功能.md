## 二、会议议程与讨论内容

### 1. 产品背景与市场分析（14:00-14:30）

张产品首先介绍了智忆助理1.0版本的用户反馈数据：
- 日活跃用户(DAU)达到12,000人
- 用户平均会话时长8.5分钟
- 记忆召回准确率达到87%
- 用户满意度评分4.2/5.0

**市场竞品分析：**
- Notion AI：强项在文档协作，弱项在个性化记忆
- Mem.ai：记忆功能强大但中文支持不佳
- 讯飞星火：通用AI能力强但缺乏长期记忆

**2.0版本核心定位：**
> 打造中文用户最贴心的个性化长期记忆助手，实现"越用越懂你"的产品体验。

---

### 2. 核心功能需求评审（14:30-16:00）

#### 2.1 智能记忆增强功能

**需求描述：**
- 支持多模态记忆（文本、图片、语音、文档）
- 记忆关联图谱可视化
- 智能记忆总结与归档

**讨论要点：**
- 王开发提出多模态需要接入CLIP模型，预计增加200ms延迟
- 周设计建议记忆图谱采用力导向图布局，支持缩放与拖拽
- 决定采用渐进式加载策略，优先保证核心功能流畅性

**关键决策：**
- ✅ 采用腾讯云TI-ONE平台进行模型部署，成本降低30%
- ✅ 记忆图谱功能作为实验性功能上线，默认关闭

#### 2.2 用户画像深度学习能力

**需求描述：**
- 自动学习用户偏好与习惯
- 个性化回复风格调整
- 领域专业知识积累

**技术方案讨论：**
- 刘后端建议使用LoRA微调技术，用户级个性化模型
- 赵前端提出需要设计用户画像可视化页面
- 陈测试关注隐私合规，需要支持数据导出与删除

**关键决策：**
- ✅ 用户画像学习采用联邦学习方案，原始数据不出端
- ✅ 增加"隐私模式"开关，用户可关闭画像学习
- ✅ 用户画像页面由赵前端负责，4月15日前完成设计稿

#### 2.3 企业版功能（付费增值）

**需求描述：**
- 团队协作空间
- 企业知识库集成
- 管理员控制台
- API开放平台

**商业模式讨论：**
- 个人版：免费，基础记忆功能
- 专业版：¥29/月，高级记忆与无限存储
- 企业版：¥99/人/月，团队协作与API接入

**关键决策：**
- ✅ 企业版功能推迟到2.1版本，2.0聚焦个人用户
- ✅ 专业版订阅功能由王开发牵头开发

### 3. 技术架构调整方案（16:00-16:45）

#### 3.1 存储架构升级

**现状问题：**
- 当前SQLite单库架构，查询性能瓶颈
- 向量索引与元数据分离，一致性问题
- 缺乏冷热数据分层

**新架构方案：**
```
┌─────────────────────────────────────────────────────┐
│                    接入层                            │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────┐  │
│  │   Web/API    │  │  WebSocket   │  │ Lark Bot │  │
│  └──────────────┘  └──────────────┘  └──────────┘  │
└─────────────────────────────────────────────────────┘
                         │
┌─────────────────────────────────────────────────────┐
│                   服务层                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────┐  │
│  │ Memory Agent │  │ RAG Engine   │  │ Profile  │  │
│  └──────────────┘  └──────────────┘  └──────────┘  │
└─────────────────────────────────────────────────────┘
                         │
┌─────────────────────────────────────────────────────┐
│                   存储层                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────┐  │
│  │  TiDB Cluster│  │ Milvus Vector│  │   Redis  │  │
│  │  (Metadata)  │  │   (Vectors)  │  │  (Cache) │  │
│  └──────────────┘  └──────────────┘  └──────────┘  │
└─────────────────────────────────────────────────────┘
```

**关键决策：**
- ✅ 数据库迁移至TiDB集群，支持水平扩展
- ✅ 向量库采用Milvus，替换FAISS
- ✅ 引入Redis缓存热数据，降低DB压力

#### 3.2 LLM服务优化

**优化措施：**
- 实现智能路由：简单查询走轻量模型，复杂任务走强模型
- 增加流式响应，首字节时间(TTFB)目标<500ms
- 建立提示词版本管理，支持A/B测试

**关键决策：**
- ✅ 接入DeepSeek-V3作为主力模型，Qwen3.5作为备选
- ✅ 提示词管理使用Langfuse平台
- ✅ 由刘后端负责LLM网关开发，4月20日提测

### 5. 关键决策汇总

**已确认决策：**

1. ✅ **技术栈升级：** 采用TiDB+Milvus+Redis新架构，放弃SQLite+FAISS
2. ✅ **模型选择：** 主力模型DeepSeek-V3，备选Qwen3.5，轻量任务GLM-4-Flash
3. ✅ **隐私保护：** 用户画像联邦学习，增加隐私模式开关
4. ✅ **版本策略：** 企业版推迟到2.1版本，2.0聚焦个人用户
5. ✅ **付费模式：** 个人版免费，专业版¥29/月，企业版推迟上线
6. ✅ **性能目标：** TTFB<500ms，端到端延迟<3s，支持1万并发

**待确认决策：**

- ⏳ 是否接入Apple Intelligence（等WWDC后决定）
- ⏳ 安卓客户端开发排期（看iOS版本反响）
- ⏳ 是否开源部分模块（需法务评估）
